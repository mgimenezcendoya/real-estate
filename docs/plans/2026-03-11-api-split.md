# API Monolith Split — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Dividir `app/admin/api.py` (4430 líneas) en 12 routers de dominio + un archivo de dependencias compartidas, sin cambiar ninguna URL ni comportamiento.

**Architecture:** Se crea `app/admin/deps.py` con los helpers compartidos (`security`, `_audit`, `_require_admin`, `_get_actor`, `ADMIN_ROLES`). Cada dominio obtiene su propio `app/admin/routers/<domain>.py` con su `APIRouter()`. El `api.py` original se convierte en un aggregator de ~30 líneas que incluye todos los routers. `main.py` no cambia en absoluto.

**Tech Stack:** FastAPI, asyncpg, Pydantic v2, Python 3.11+

**Regla de oro:** Cada router file es una extracción directa de api.py — copy/paste inteligente, cero lógica nueva. Si algo funciona hoy, debe funcionar igual después.
